# <a name="always-on-vpn-features-and-functionalities"></a>Recursos e funcionalidades de VPN Always On

>Aplicável a: Windows Server (Canal Semestral), Windows Server 2016, Windows 10

- [**Anterior:** Implantação de VPN Always On para Windows Server e Windows 10](always-on-vpn/deploy/always-on-vpn-deploy.md)
- [**Em seguida:** Saiba mais sobre os aprimoramentos de VPN Always On](../vpn/always-on-vpn/always-on-vpn-enhancements.md)

Neste tópico, você aprenderá sobre os recursos e funcionalidades do Always On VPN.  A tabela a seguir não é uma lista completa, no entanto, inclui alguns dos recursos e funcionalidades mais comuns usados nas soluções de acesso remoto. 

> [!TIP]
> Se você usa atualmente o DirectAccess, recomendamos que investigue a funcionalidade de VPN Always On cuidadosamente para determinar se ela atende a todas as suas necessidades de acesso remoto antes de migrar do DirectAccess para Always On VPN.  

|                                                   Área funcional                                                   |                                                                                                                                                                                                                                                                                                                                                                                                                                                     VPN Always On                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|---------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|                            Conectividade direta e transparente com a rede corporativa.                             |                                                                                                                                                                                                                                                                                                                  Você pode configurar Always On VPN para dar suporte ao disparo automático com base nas solicitações de inicialização do aplicativo ou de resolução de namespace.<p><p>Definir usando:<br>**VPNv2/ProfileName/AlwaysOn**<br>**VPNv2/ProfileName/AppTriggerList**<br>**VPNv2/ProfileName/DomainNameInformationList/gatilho autotrigger**                                                                                                                                                                                                                                                                                                                  |
|  Uso de um túnel de infraestrutura dedicado para fornecer conectividade para os usuários que não entraram na rede corporativa.  |                                                                                                                                                                                                                                                                                                               Você pode obter essa funcionalidade usando o recurso de túnel de dispositivo no perfil de VPN.<p><p>***Observação:***<br>O túnel de dispositivo só pode ser configurado em dispositivos ingressados no domínio usando IKEv2 com autenticação de certificado do computador.<p><p>Definir usando:<br>**VPNv2/ProfileName/DeviceTunnel**                                                                                                                                                                                                                                                                                                               |
| Uso do gerenciamento para permitir a conectividade remota com clientes de sistemas de gerenciamento localizados na rede corporativa. |                                                                                                                                                                                                   Você pode obter essa funcionalidade usando o recurso de encapsulamento de dispositivo no perfil VPN combinado com a configuração da conexão VPN para registrar dinamicamente os endereços IP atribuídos à interface VPN com serviços DNS internos.<p><p>***Observação:***<br>Se você ativar os filtros de tráfego no perfil de túnel de dispositivo, o túnel de dispositivo negará o tráfego de entrada (da rede corporativa para o cliente).<p><p>Definir usando:<br>**VPNv2/ProfileName/DeviceTunnel**<br>**VPNv2/ProfileName/RegisterDNS**                                                                                                                                                                                                    |
|                            Volte quando os clientes estiverem protegidos por firewalls ou servidores proxy.                            |                                                                                                                                                                                                                                                                                          Você pode configurar o para fazer fallback para SSTP (de IKEv2) usando o tipo de túnel/protocolo automático dentro do perfil de VPN.<p><p>***Observação:***<br>O túnel do usuário dá suporte a SSTP e IKEv2, e o túnel de dispositivo dá suporte a IKEv2 somente sem suporte para fallback de SSTP.<p>Definir usando:<br>**VPNv2/ProfileName/NativeProfile/NativeProtocolType**                                                                                                                                                                                                                                                                                           |
|                                        Suporte para o modo de acesso de ponta a ponta.                                         |                                                                                                                                                                                                                                                                                                       Always On VPN fornece conectividade com os recursos corporativos usando políticas de túnel que exigem autenticação e criptografia até alcançarem o gateway de VPN. Por padrão, as sessões de túnel são encerradas no gateway de VPN, que também funciona como o gateway IKEv2, fornecendo segurança de ponta a ponta.                                                                                                                                                                                                                                                                                                        |
|                                   Suporte para autenticação de certificado do computador.                                   |                                                                                                                                                                                                                                                                  O tipo de protocolo IKEv2 disponível como parte da plataforma de VPN Always On dá suporte especificamente ao uso de certificados de computador ou máquina para autenticação de VPN.<p><p>***Observação:***<br>IKEv2 é o único protocolo com suporte para o túnel de dispositivo e não há nenhuma opção de suporte para fallback de SSTP. <p>Definir usando:<br>**VPNv2/ProfileName/NativeProfile/autenticação/MachineMethod**                                                                                                                                                                                                                                                                   |
|                    Use grupos de segurança para limitar a funcionalidade de acesso remoto a clientes específicos.                    |                                                                                                                                                                                                                                                                                                                                                                                 Você pode configurar Always On VPN para dar suporte à autorização granular ao usar o RADIUS, que inclui o uso de grupos de segurança para controlar o acesso VPN.                                                                                                                                                                                                                                                                                                                                                                                  |
|                             Suporte para servidores atrás de um firewall de borda ou dispositivo NAT.                              |                                                                                                                                                                                                                                                                                                          Always On VPN oferece a capacidade de usar protocolos como IKEv2 e SSTP que dão suporte total ao uso de um gateway de VPN que está atrás de um dispositivo NAT ou de um firewall de borda.<p><p>***Observação:***<br>O túnel do usuário dá suporte a SSTP e IKEv2, e o túnel de dispositivo dá suporte a IKEv2 somente sem suporte para fallback de SSTP.                                                                                                                                                                                                                                                                                                          |
|                 Capacidade de determinar a conectividade da intranet quando conectada à rede corporativa.                 |                                                                                                                                                                                                                                                                                                               A detecção de rede confiável fornece a capacidade de detectar conexões de rede corporativa e é baseada em uma avaliação do sufixo DNS específico da conexão atribuído às interfaces de rede e ao perfil de rede.<p><p>Definir usando:<br>**VPNv2/ProfileName/TrustedNetworkDetection**                                                                                                                                                                                                                                                                                                                |
|                                  Conformidade usando a NAP (proteção de acesso à rede).                                  | O cliente VPN Always On pode se integrar com o acesso condicional do Azure para impor MFA, conformidade do dispositivo ou uma combinação de ambos. Quando em conformidade com as políticas de acesso condicional, o Azure AD emite um certificado de autenticação IPsec de curta duração (por padrão, 60 minutos) que o cliente pode usar para se autenticar no gateway de VPN. A conformidade do dispositivo aproveita as políticas de conformidade do Configuration Manager/Intune, que podem incluir o estado do atestado de integridade do dispositivo. Neste momento, o acesso condicional de VPN do Azure fornece a substituição mais próxima à solução de NAP existente, embora não haja nenhuma forma de serviço de correção ou recursos de rede de quarentena. Para obter mais detalhes, consulte [VPN e acesso condicional](https://docs.microsoft.com/windows/security/identity-protection/vpn/vpn-conditional-access).<p>Definir usando:<br>**VPNv2/ProfileName/DeviceCompliance** |
|                   Capacidade de definir quais servidores de gerenciamento podem ser acessados antes de entrar no usuário.                    |                                                                                                                                                                        Você pode obter essa funcionalidade no Always On VPN usando o recurso de encapsulamento de dispositivo (disponível na versão 1709 – somente para IKEv2) no perfil VPN combinado com filtros de tráfego para controlar quais sistemas de gerenciamento na rede corporativa podem ser acessados por meio do Túnel de dispositivo.<p><p>***Observação:***<br>Se você ativar os filtros de tráfego no perfil de túnel de dispositivo, o túnel de dispositivo negará o tráfego de entrada (da rede corporativa para o cliente).<p>Definir usando:<br>**VPNv2/ProfileName/DeviceTunnel**<br>**VPNv2/ProfileName/TrafficFilterList**                                                                                                                                                                         |

## <a name="additional-functionalities"></a>Funcionalidades adicionais

Cada item nesta seção é um cenário de caso de uso ou a funcionalidade de acesso remoto usada com frequência para a qual Always On VPN melhorou a funcionalidade — seja por meio de uma expansão de funcionalidade ou eliminação de uma limitação anterior.


|                                               Área funcional                                                |                                                                                                                                                                                                                                                                                               VPN Always On                                                                                                                                                                                                                                                                                               |
|--------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|                           Dispositivos ingressados no domínio com requisito de SKUs empresariais.                            |                                           Always On VPN dá suporte a dispositivos ingressados no domínio, não ingressados no domínio (grupo de trabalho) ou ao Azure AD – para permitir cenários empresariais e BYOD. Always On VPN está disponível em todas as edições do Windows, e os recursos de plataforma estão disponíveis para terceiros por meio do suporte de plug-in de VPN do UWP.<p><p>***Observação:***<br>O túnel de dispositivo só pode ser configurado em dispositivos ingressados no domínio que executam o Windows 10 Enterprise ou o Education versão 1709 ou posterior. Não há suporte para controle de terceiros do túnel do dispositivo.                                           |
|                                       Suporte para IPv4 e IPv6.                                        |                                                                                                                                                                 Com a VPN Always On, os usuários podem acessar os recursos de IPv4 e IPv6 na rede corporativa. O cliente VPN Always On usa uma abordagem de pilha dupla que não depende especificamente do IPv6 ou da necessidade de o gateway de VPN fornecer serviços de tradução NAT64 ou DNS64.                                                                                                                                                                  |
|                                Suporte para autenticação de dois fatores ou OTP.                                 |                    O Always On plataforma de VPN nativamente dá suporte ao EAP, que permite o uso de diversos tipos de EAP da Microsoft e de terceiros como parte do fluxo de trabalho de autenticação. Always On VPN especificamente dá suporte a certificados de cartão inteligente (físico e virtual) e do Windows Hello para empresas para atender aos requisitos de autenticação de dois fatores. Além disso, Always On VPN dá suporte à OTP por meio da MFA (sem suporte nativamente, só tem suporte em plug-ins de terceiros) por meio da integração RADIUS do EAP.<p><p>Definir usando:<br>**VPNv2/ProfileName/NativeProfile/autenticação**                    |
|                                  Suporte para vários domínios e florestas.                                   |                                   A plataforma de VPN Always On não tem dependências de florestas de Active Directory Domain Services (AD DS) ou topologia de domínio (ou níveis funcionais/de esquema associados) porque não exige que o cliente VPN seja ingressado no domínio para funcionar. O Política de Grupo, portanto, não é uma dependência para definir as configurações de perfil de VPN porque você não a usa durante a configuração do cliente. Onde Active Directory integração de autorização é necessária, você pode obtê-la por meio do RADIUS como parte do processo de autenticação e autorização EAP.                                   |
|              Suporte para o túnel dividido e forçado para separação de tráfego de Internet/intranet.               |                                                                                               Você pode configurar Always On VPN para dar suporte ao túnel de força (o modo de operação padrão) e ao túnel dividido nativamente. Always On VPN fornece granularidade adicional para políticas de roteamento específicas do aplicativo.<p><p>***Observação:***<br>O túnel forçado tem suporte somente pelo túnel do usuário. Nenhum outro túneI de VPN pode estar ativo em paralelo a um túnel de usuário de túnel forçado.<p><p>Definir usando:<p> **VPNv2/ProfileName/NativeProfile/RoutingPolicyType**<br>**VPNv2/ProfileName/TrafficFilterList/app/RoutingPolicyType**                                                                                               |
|                                          Suporte a vários protocolos.                                          |                                                                                                                                                                            Always On VPN pode ser configurada para dar suporte ao SSTP nativamente se protocolo SSL fallback de IKEv2 for necessário.<p><p>***Observação:***<br>O túnel do usuário dá suporte a SSTP e IKEv2, e o túnel de dispositivo dá suporte a IKEv2 somente sem suporte para fallback de SSTP.                                                                                                                                                                            |
|                       Assistente de conectividade para fornecer o status de conectividade corporativa.                       |                                                                                                            Always On VPN está totalmente integrada ao assistente de conectividade de rede nativo e fornece o status de conectividade da interface exibir todas as redes. Com o advento da atualização do Windows 10 para criadores (versão 1703), o status da conexão VPN e o controle de conexão VPN para o túnel do usuário agora estão disponíveis por meio do submenu da rede (para o cliente VPN interno do Windows) também.                                                                                                            |
| Resolução de nomes de recursos corporativos usando o FQDN (nome de domínio totalmente qualificado) e o sufixo DNS de nome curto. | Always On VPN pode definir nativamente um ou mais sufixos DNS como parte da conexão VPN e do processo de atribuição de endereço IP, incluindo a resolução de nomes de recursos corporativos para nomes curtos, FQDNs ou namespaces DNS inteiros. Always On VPN também dá suporte ao uso de tabelas de política de resolução de nomes para fornecer granularidade de resolução específica de namespace.<p><p>***Observação:***<br>Evite o uso de sufixos globais, pois eles interferem na resolução de curtoname ao usar tabelas de política de resolução de nomes.<p><p>Definir usando:<br>**VPNv2/ProfileName/DnsSuffix**<br>**VPNv2/ProfileName/DomainNameInformationList** |
