# <a name="directaccess-deployment-paths-in-windows-server"></a>Caminhos de implantação do DirectAccess no Windows Server

>Aplica-se a: Windows Server (canal semestral), Windows Server 2016

Este tópico fornece uma listagem da documentação para os dois caminhos principais de implantação do acesso remoto: **Básica** e **Advanced**.  
  
Você pode usar a seção a seguir para obter uma compreensão das diferenças entre os DirectAccess Basic e os caminhos de implantação avançada, e você pode usar os links de documentação para localizar o guia de implantação é mais adequado para suas metas.  
  
## <a name="deploy-basic-directaccess"></a>Implantar o DirectAccess básico  
Com uma implantação básica do DirectAccess, o DirectAccess é configurado com as configurações padrão usando um assistente, sem qualquer necessidade de definir as configurações de infraestrutura, como uma autoridade de certificação (CA) ou grupos de segurança do Active Directory.  
  
-   [Implantar um único servidor DirectAccess usando o Assistente de Introdução](../../remote-access/directaccess/single-server-wizard/Deploy-a-Single-DirectAccess-Server-Using-the-Getting-Started-Wizard.md). Você pode usar este guia para implantar o DirectAccess básico em um ambiente de produção.  
  
## <a name="deploy-advanced-directaccess"></a>Implantar o DirectAccess avançado  
Com uma implantação avançada do DirectAccess, você pode implantar um único servidor DirectAccess e configurar servidores de infraestrutura de rede para dar suporte a DirectAccess.  
  
-   [Implantar um único servidor DirectAccess com configurações avançadas](../../remote-access/directaccess/single-server-advanced/Deploy-a-Single-DirectAccess-Server-with-Advanced-Settings.md). Você pode usar este guia para implantar o DirectAccess com configurações avançadas em um ambiente de produção.
