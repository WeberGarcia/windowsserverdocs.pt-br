# <a name="manage-role-based-access-control-with-server-manager"></a>Gerenciar baseado em função com o Gerenciador do servidor de controle de acesso

>Aplica-se a: Windows Server (anual por canal), Windows Server 2016

Você pode usar os seguintes tópicos para gerenciar o controle de acesso baseado na função usando o Gerenciador do servidor, que tem uma interface gráfica do usuário.  
  
-   [Crie uma função de usuário para o controle de acesso](../../technologies/ipam/Create-a-User-Role-for-Access-Control.md)  
  
-   [Criar uma política de acesso](../../technologies/ipam/Create-an-Access-Policy.md)  
  
-   [Defina o escopo de acesso para uma zona de DNS](../../technologies/ipam/Set-Access-Scope-for-a-DNS-Zone.md)
  
-   [Defina o escopo de acesso para registros de recurso do DNS](../../technologies/ipam/Set-Access-Scope-for-DNS-Resource-Records.md)
  
-   [Modo de exibição de funções e permissões da função](../../technologies/ipam/View-Roles-and-Role-Permissions.md)
  
Como alternativa, você pode usar o Windows PowerShell para gerenciar o controle de acesso baseado na função IPAM. Para obter mais informações, consulte [gerenciar função com base em controle de acesso com o Windows PowerShell](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Windows-PowerShell.md).
