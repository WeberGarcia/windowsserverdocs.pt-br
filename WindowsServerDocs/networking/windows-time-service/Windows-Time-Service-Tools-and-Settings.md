# <a name="windows-time-service-tools-and-settings"></a>Ferramentas e configurações do Serviço de Tempo do Windows
>Aplica-se a: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows 10 ou posterior

Neste tópico, você aprenderá sobre as ferramentas e configurações para o serviço de tempo do Windows (W32Time). 

Se você deseja sincronizar a hora para um computador cliente ingressado no domínio, consulte [configurar um computador cliente para sincronização de hora automática domínio](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2008-r2-and-2008/cc816884%28v%3dws.10%29). Para tópicos adicionais sobre como configurar o serviço de tempo do Windows, consulte [onde encontrar informações de configuração serviço de tempo de Windows](https://docs.microsoft.com/windows-server/networking/windows-time-service/windows-time-service-top).  

> [!CAUTION]  
> Você não deve usar o comando Net time para configurar ou definir quando o serviço de tempo do Windows estiver em execução.  
>
> Além disso, em computadores mais antigos que executam o Windows XP ou versões anteriores, o comando Net time /querysntp exibe o nome de um servidor de protocolo NTP (Network Time) com o qual um computador é configurado para sincronizar, mas esse servidor NTP é usado somente quando o cliente do computador tempo é configurado como NTP ou AllSync. Esse comando já foi preterido.  

A maioria dos computadores de membros do domínio tem um tipo de cliente do tempo do NT5DS, o que significa que eles sincronizem o horário da hierarquia de domínio. A exceção apenas típica é o controlador de domínio que funciona como o mestre de operações do emulador PDC (controlador) domínio primário do domínio raiz da floresta, que geralmente é configurado para sincronizar a hora com uma fonte de tempo externa. Para exibir a configuração de cliente de tempo de um computador, execute o comando a W32tm /query/configuração em um Prompt de comando elevado na partir do Windows Server 2008 e Windows Vista e leia as **tipo** linha na saída do comando. Para obter mais informações, consulte [funciona com o Windows tempo de serviço como](https://docs.microsoft.com/windows-server/networking/windows-time-service/How-the-Windows-Time-Service-Works). Você pode executar o comando **reg query HKLM\SYSTEM\CurrentControlSet\Services\W32Time\Parameters** e ler o valor de **NtpServer** na saída do comando.  

> [!IMPORTANT]  
> Antes do Windows Server 2016, o serviço W32Time não foi projetado para atender às necessidades do aplicativo sensível ao tempo.  No entanto, as atualizações para o Windows Server 2016 agora permitem que você implemente uma solução para 1ms precisão em seu domínio.  Ver [tempo do Windows 2016 precisos](accurate-time.md) e [limite de suporte para configurar o serviço de tempo do Windows para ambientes de alta precisão](https://docs.microsoft.com/windows-server/networking/windows-time-service/support-boundary) para obter mais informações.

As ferramentas a seguir estão associadas com o serviço de tempo do Windows.  

#### <a name="w32tmexe-windows-time"></a>W32tm.exe: Tempo do Windows  
**categoria**  

Essa ferramenta é instalada como parte do Windows XP, Windows Vista, Windows 7, Windows Server 2003, Windows Server 2003 R2, Windows Server 2008 e instalações padrão do Windows Server 2008 R2.  

**Compatibilidade de versão**  

Essa ferramenta funciona no Windows XP, Windows Vista, Windows 7, Windows Server 2003, Windows Server 2003 R2, Windows Server 2008 e instalações padrão do Windows Server 2008 R2.  

W32tm.exe é usado para definir as configurações de serviço de tempo do Windows. Ele também pode ser usado para diagnosticar problemas com o serviço de tempo. W32tm.exe é a ferramenta de linha de comando preferido para configuração, monitoramento ou o serviço de tempo do Windows de solução de problemas.

## <a name="windows-time-service-registry-entries"></a>Entradas de registro do serviço de tempo do Windows
As seguintes entradas do registro estão associadas com o serviço de tempo do Windows.  

Essas informações são fornecidas como uma referência para uso na solução de problemas ou verificando que as configurações necessárias sejam aplicadas. É recomendável que você não editar diretamente o registro, a menos que não há nenhuma outra alternativa. Modificações no registro não são validadas pelo editor do registro ou pelo Windows antes que eles são aplicados e como resultado, os valores incorretos podem ser armazenados. Isso pode resultar em erros irrecuperáveis no sistema.  

Quando possível, use a diretiva de grupo ou outras ferramentas do Windows, como o Microsoft Management Console (MMC), para realizar tarefas em vez de editar o registro diretamente. Se você deve editar o Registro, tenha muito cuidado.  

> [!WARNING]  
> Alguns dos valores predefinidos que são configurados no arquivo de modelo administrativo do sistema (ADM) para as configurações de GPO (objeto) de diretiva de grupo são diferentes das entradas de registro de padrão correspondentes. Se você planeja usar um GPO para definir uma configuração de tempo do Windows, certifique-se de que você examine [valores de predefinição para as configurações de diretiva de grupo de serviço de tempo do Windows são diferentes de entradas de registro de serviço de tempo do Windows correspondentes no Windows Server 2003 ](https://go.microsoft.com/fwlink/?LinkId=186066). Esse problema se aplica ao Windows Server 2008 R2, Windows Server 2008, Windows Server 2003 R2 e Windows Server 2003.  

Muitas entradas do registro para o serviço de tempo do Windows são o mesmo que a configuração de diretiva de grupo de mesmo nome. As configurações de diretiva de grupo correspondem às entradas do registro do mesmo nome localizado em:  

>**HKLM\SYSTEM\CurrentControlSet\Services\W32Time\\**


Há várias chaves do registro no local do registro. As configurações de tempo do Windows são armazenadas nos valores em todas essas chaves:

* [Parâmetros](#hklmsystemcurrentcontrolsetservicesw32timeparameters)
* [Config](#hklmsystemcurrentcontrolsetservicesw32timeconfig)
* [NtpClient](#hklmsystemcurrentcontrolsetservicesw32timetimeprovidersntpclient)
* [NtpServer](#hklmsystemcurrentcontrolsetservicesw32timetimeprovidersntpserver)

Muitos dos valores na seção W32Time do registro são usados internamente pelo W32Time para armazenar informações. Esses valores não devem ser alterados manualmente a qualquer momento. Não modifique as configurações nesta seção, a menos que você esteja familiarizado com a configuração e tiver certeza de que o novo valor funcionará conforme o esperado. As seguintes entradas do registro estão localizadas em:

**HKLM\SYSTEM\CurrentControlSet\Services\W32Time**  

Quando você cria uma política, as definições são configuradas no local a seguir, que tem precedência sobre o próximo local:

**HKLM\SOFTWARE\Policies\Microsoft\Windows\W32time**

A chave W32time é criada com a política.  Quando você remove a política, essa chave também é removida.

O outro local padrão:

**HKLM\SYSTEM\CurrentControlSet\Services\W32time**

Alguns dos parâmetros são armazenados em tiques do relógio no registro e alguns são em segundos. Para converter a hora de tiques do relógio em segundos:  

-   1 minuto = 60 s  

-   1 sec = 1000 ms  

-   1 ms = 10.000 tiques do relógio em um sistema Windows, conforme descrito em [ticks Property](https://docs.microsoft.com/dotnet/api/system.datetime.ticks?redirectedfrom=MSDN&view=netframework-4.7.2#System_DateTime_Ticks).  

Por exemplo, 5 minutos tornaria 5\*60\*1000\*3000000000 = 10000 tiques do relógio. 

Todas as versões incluem Windows 7, Windows 8, Windows 10, Windows Server 2008 e Windows Server 2008 R2, Windows Server 2012, Windows Server 2012 R2, Windows Server 2016.  Algumas entradas só estão disponíveis em versões mais recentes do Windows.

#### <a name="maxallowedphaseoffset-information"></a>Informações de MaxAllowedPhaseOffset
Para W32Time definir o relógio do computador gradualmente, o deslocamento deve ser menor do que o **MaxAllowedPhaseOffset** de valor e satisfaz a seguinte equação ao mesmo tempo:  

```  
|CurrentTimeOffset| / (PhaseCorrectRate*UpdateInterval) < SystemClockRate / 2  
``` 
O CurrentTimeOffset é medido em tiques do relógio, em que 1 ms = 10.000 tiques em um sistema Windows do relógio.  

SystemClockRate e PhaseCorrectRate também são medidos em tiques do relógio. Para obter o SystemClockRate, você pode usar o comando a seguir e convertê-lo de segundos para usando a fórmula de segundos de tiques do relógio * 1000\*10000:  

```  
W32tm /query /status /verbose  
ClockRate: 0.0156000s  
```  

SystemclockRate é a velocidade do relógio do sistema. Usando 156000 segundos como um exemplo, o SystemclockRate poderia ser = 0.0156000 \* 1000 \* 156000 = 10000 tiques do relógio.  

MaxAllowedPhaseOffset também está em segundos. Para convertê-los para tiques do relógio, multiplique MaxAllowedPhaseOffset * 1000\*10000.  

Os dois exemplos a seguir mostram como aplicar  

**Exemplo 1**: Horário difere em 4 minutos (por exemplo, o tempo é de 11 12:05, e o exemplo de hora recebida de um par e acredita-se que seja correta é 11:09 AM).
  
```
phasecorrectRate = 1  

UpdateInterval = 30000 (clock ticks)  

systemclockRate = 156000 (clock ticks)  

MaxAllowedPhaseOffset = 10min = 600 seconds = 600*1000\*10000=6000000000 clock ticks  

|currentTimeOffset| = 4mins = 4*60\*1000\*10000 = 2400000000 ticks  

Is CurrentTimeOffset < MaxAllowedPhaseOffset?  

2400000000 < 6000000000 = TRUE  
```

E ele atende a equação acima? 

```
(|CurrentTimeOffset| / (PhaseCorrectRate*UpdateInterval) < SystemClockRate / 2)  

Is 2,400,000,000 / (30000*1) < 156000/2  

Is 80,000 < 78,000  

NO/FALSE  
```  

Portanto W32tm definiria o relógio volta imediatamente.  

> [!NOTE]  
> Nesse caso, se você quiser configurar o relógio volta lentamente, você precisaria ajustar os valores de PhaseCorrectRate ou updateInterval no registro também para garantir que os resultados da equação em TRUE.  

**Exemplo 2**: Horário difere em 3 minutos. 
 
```  
phasecorrectRate = 1  

UpdateInterval = 30000 (clock ticks)  

systemclockRate = 156000 (clock ticks)  

MaxAllowedPhaseOffset = 10min = 600 seconds = 600*1000\*10000=6000000000 clock ticks  

currentTimeOffset = 3mins = 3*60\*1000\*10000 = 1800000000 clock ticks  

Is CurrentTimeOffset < MaxAllowedPhaseOffset?  

1800000000 < 6000000000 = TRUE  
```  

E ele atende a equação acima?

```
(|CurrentTimeOffset| / (PhaseCorrectRate*UpdateInterval) < SystemClockRate / 2)  

Is 3 mins (1,800,000,000) / (30000*1) < 156000/2  

Is 60,000 < 78,000  

YES/TRUE  
```  

Nesse caso, o relógio será definido lentamente.

## <a name="network-ports-used-by-the-windows-time-service"></a>Portas de rede usadas pelo serviço de tempo do Windows  
Tempo do Windows segue a especificação de NTP, que requer o uso da porta UDP 123 para todas as comunicações de sincronização de tempo. Essa porta é reservada pelo Windows tempo e permanece reservada em todos os momentos. Sempre que o computador sincroniza seu relógio ou dá tempo para outro computador, essa comunicação é realizada na porta UDP 123.  

> [!NOTE]  
> Se você tiver um computador com vários adaptadores de rede (também chamados de um computador multihomed), será possível habilitar seletivamente o serviço de tempo do Windows com base no adaptador de rede.
