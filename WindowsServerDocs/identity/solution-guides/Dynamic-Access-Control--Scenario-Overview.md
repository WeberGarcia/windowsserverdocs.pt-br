# <a name="dynamic-access-control-scenario-overview"></a>Controle de acesso dinâmico: Visão geral do cenário

>Aplica-se a: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

No Windows Server 2012, você pode aplicar governança de dados em seus servidores de arquivos para controlar quem pode acessar as informações e de auditoria que teve acesso a informações. Controle de acesso dinâmico permite que você:  
  
-   Identifique dados usando classificação automática e manual de arquivos. Por exemplo, você pode marcar dados em servidores de arquivos em toda a organização.  
  
-   Controlar o acesso a arquivos pela aplicação de políticas de rede de segurança que usam políticas de acesso central. Por exemplo, você pode definir quem pode acessar informações de saúde dentro da organização.  
  
-   Auditoria de acesso a arquivos usando políticas de auditoria central para análise de relatório e forenses de conformidade. Por exemplo, você pode identificar quem acessou informações altamente confidenciais.  
  
-   Aplica proteção Rights Management Services (RMS) usando criptografia do RMS automática para documentos do Microsoft Office confidenciais. Por exemplo, você pode configurar o RMS para criptografar todos os documentos que contêm informações sobre portabilidade de seguros de saúde e responsabilidade Act (HIPAA).  
  
O conjunto de recursos de controle de acesso dinâmico se baseia investimentos de infraestrutura que podem ser aplicativos mais usados por parceiros e de linha de negócios e os recursos podem fornecer valor excelente para organizações que usam o Active Directory. Essa infraestrutura inclui:  
  
-   Um mecanismo de autorização e auditoria novo para o Windows que pode processar expressões condicionais e políticas central.  
  
-   Suporte de autenticação Kerberos para declarações do usuário e declarações de dispositivo.  
  
-   Melhorias para a infraestrutura de classificação de arquivo (FCI).  
  
-   Suporte de extensibilidade do RMS para que os parceiros possam fornecer soluções que criptografar arquivos não são da Microsoft.
