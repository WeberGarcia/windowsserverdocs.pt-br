# <a name="ad-forest-recovery---devising-an-ad-forest-recovery-plan"></a>Plano de recuperação recuperação de floresta do AD - descobrindo um anúncio da floresta

>Aplica-se a: Windows Server 2016, Windows Server 2012 e 2012 R2, Windows Server 2008 e 2008 R2

Dependendo do seu ambiente e requisitos de negócios, você pode ou não talvez seja necessário executar todas as etapas descritas neste guia para realizar uma recuperação de floresta bem-sucedida. Considerando que este guia serve apenas como um modelo para a recuperação de floresta, é essencial que você planejar um plano de recuperação de floresta personalizado que se adapte ao seu ambiente e atenda às suas necessidades de negócios.  
  
 Por exemplo, no seu plano de recuperação de floresta, você deve ter um mapa de seu florestas topologia detalhadas. O mapa deve listar todas as informações sobre os controladores de domínio, como seus nomes, suas funções e status do backup e as relações de confiança entre eles. Para uma ferramenta que você pode usar para criar um mapa de topologia, consulte [Microsoft Active Directory topologia Diagrammer](https://www.microsoft.com/download/details.aspx?id=13380).  
  
 Você deve praticar seu plano de recuperação de floresta pelo menos uma vez por ano. Além disso, é uma boa ideia para executar uma análise de recuperação de floresta quando há alterações de associação ao grupo Administradores corporativos ou administradores de domínio. Isso ajuda a garantir que sua equipe de informática informação totalmente entenda o plano de recuperação de floresta.
